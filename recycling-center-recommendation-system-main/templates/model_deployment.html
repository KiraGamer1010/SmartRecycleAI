CRISP-ML lifecycle

Model Deployment

Model deployment explains how a validated SmartRecycleAI model would move from academic experimentation into a reliable operational environment for recycling center recommendation and municipal decision support.

Operational readiness

From validated model to production service

Production Implementation

A validated model would be integrated into the Flask platform and deployed on Render.com through the existing WSGI entry point. The current web interface can present recommendations, while future API endpoints can expose predictions to institutional systems or municipal dashboards.

Documentation and Training

Deployment requires technical documentation, user guidance, feature definitions, model limitations, and training material for staff who interpret recommendation outputs. Clear documentation helps avoid misuse and supports reproducible decision making.

Protocol Establishment

Production use should define monitoring procedures, incident response, change management, feedback collection, model approval criteria, and escalation paths when recommendations conflict with field field observations or policy constraints.

Render deployment plan

Production workflow and configuration

SmartRecycleAI remains compatible with Render.com because it uses a standard Flask WSGI entry point, dependency file, Procfile, and Render service definition. Deployment does not require a separate heavy ML server; the trained model artifacts and generated static visuals are served by the same application.

Deployment item Project file or behavior Purpose
WSGI entry point wsgi.py Exposes the Flask application to a production server.
Build command Install dependencies and generate required datasets or artifacts when configured. Prepares the environment before the web service starts.
Start command Gunicorn or the configured Procfile command. Runs Flask in a production-compatible process.
Model artifacts models/best_model.pkl and related classifier files. Allow Prediction System inference without retraining during each request.
Static charts static/generated/ Provides visual outputs for analytical pages and report captures.

Deployment Validation

A deployment is valid when the home page, CRISP-ML pages, Model Engineering, Model Evaluation, Prediction System, K-Means pages, and supporting documentation return HTTP 200 and the prediction form can produce a result.

Limitations and Maintenance

Render deployment should be monitored for artifact availability, dependency compatibility, model file loading, static chart paths, and memory use. Future production versions may add environment variables for secure configuration, logs, and external data refresh schedules.
