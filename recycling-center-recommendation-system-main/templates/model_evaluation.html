Evaluation objective

Why model evaluation matters

Model Evaluation determines whether the trained classifiers are reliable enough to support the Prediction System. A model cannot be selected only because it trains successfully; it must be tested on records that were not used for fitting, compared with multiple metrics, and interpreted in relation to the recycling planning problem.

Avoid Data Leakage

Evaluation uses the held-out test split, preserving separation between training and final metric reporting.

Compare Tradeoffs

Accuracy, precision, recall, F1-score, ROC curves, and confusion matrices reveal different model behaviors.

Select Best Model

Random Forest is selected because it achieves the strongest weighted F1-score and robust overall metrics.

Technical note

Regression Metrics: MAE, MSE, RMSE, and R²

Not primary for this task

SmartRecycleAI's current supervised task is classification, because the model predicts a municipal priority class. Therefore, classification metrics are the primary evaluation metrics. Regression metrics are included here only as a technical reference.

MAE

Mean Absolute Error measures the average absolute difference between numeric predictions and real numeric values.

MSE

Mean Squared Error squares errors before averaging them, giving stronger penalty to large numeric mistakes.

RMSE

Root Mean Squared Error returns the squared-error measure to the original target unit.

R²

R² explains how much variance in a numeric target is explained by a regression model.

Performance Analysis

Technical explanation

Metric interpretation

Accuracy provides the global correctness view, while precision and recall explain the reliability and coverage of each priority assignment. F1-score is the most useful selection signal because it balances both types of error.

Confusion matrices should be reviewed before field use because assigning a municipality to a lower priority class can delay recycling center intervention.

Model behavior

Logistic Regression provides a transparent baseline, Decision Tree adds rule-based nonlinearity, and Random Forest improves stability by combining many trees.

The evaluation page uses only test-set results, keeping the dashboard aligned with CRISP-ML validation principles.

Limitations of Evaluation

Risk awareness

The evaluation results are strong, but they remain dependent on the current dataset, target construction, feature quality, and public data update cycle. The model should be reviewed with field experts before real investment decisions, especially because false low-priority predictions could delay infrastructure action.

Future evaluation should include additional temporal validation, external expert labels, fairness review, sensitivity analysis, and monitoring after deployment.
