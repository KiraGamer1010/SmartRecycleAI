Dataset split and training process

Reproducible supervised learning design

The target variable is {{ supervised_modeling.dataset_split.target_column }}, which represents the municipality priority class for recycling center intervention. The dataset is split with {{ supervised_modeling.dataset_split.test_size * 100 }}% reserved for testing and a reproducible random state of {{ supervised_modeling.dataset_split.random_state }}. Stratification preserves the class distribution across training and test data.

Feature Matrix

Training uses the existing `model_feature_matrix.csv` generated by the data engineering pipeline. The selected variables represent demand, accessibility, vulnerability, infrastructure, and environmental pressure.

Validation Split

Only the training partition is used for fitting preprocessing steps and model parameters. Metrics shown on this page are calculated from the held-out test partition.

Artifact Strategy

Every fitted pipeline is stored as a reusable artifact, and the best model is copied to a stable `best_model.pkl` file for the prediction system.
