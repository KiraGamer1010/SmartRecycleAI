Operational purpose

From evaluated model to usable prediction workflow

The Prediction System demonstrates the final CRISP-ML transition from code to an interactive decision support page. It loads the selected model artifact, applies the same preprocessing pipeline used during training, validates user inputs, and returns a class prediction with probabilities and confidence.

Artifact Loading

The page uses the stored best model pipeline rather than rebuilding model logic inside the template.

Preprocessing Consistency

Submitted values are mapped to the same feature order and transformations used during training.

Required system section 2

Backend Prediction Logic

Reusable pipeline

Operational model

The selected model is {{ prediction_form.model_name }}. The backend validates submitted values, converts them into the same feature order used during training, and sends them into the saved scikit-learn pipeline.

Because preprocessing is stored with the model artifact, the prediction workflow uses the same imputation, scaling, and classifier configuration developed during Model Engineering.

Academic workflow

User input is treated as one new municipality profile. The backend generates a priority class, probability distribution, and confidence score, then returns them to this Flask page for visual interpretation.

The result is decision support for recycling planning, not an automatic replacement for institutional judgment.
