Pipeline architecture

From raw files to reusable ML artifacts

The data pipeline is the technical bridge between CRISP-ML Data Engineering and every model page. It reads raw repository datasets, standardizes them, creates engineered features, exports processed files, generates charts, and prepares the feature matrix consumed by both modeling and prediction workflows.

Pipeline layer Action Generated output
Raw data ingestion Reads CSV, XLS, and XLSX files from the repository datasets folder. Dataset inventory and source profiles.
Cleaning Parses numbers, dates, text keys, facility status, and municipal identifiers. Quality summaries, missing value charts, and cleaned tables.
Feature engineering Builds access, demand, pressure, growth, capacity, and vulnerability features. Model-ready feature matrix and feature tags.
Artifact generation Writes processed CSV files, generated charts, model reports, and reusable context. Static visuals, reports, downloads, and Flask dashboard context.

Reproducibility

The same pipeline powers the home dashboard, Data Understanding, Data Engineering, Dataset Analysis, K-Means Clustering, Model Engineering, Model Evaluation, and Prediction System pages. This avoids separate manual calculations across the application.

Model-ready Dataset

The final feature matrix includes consistent numerical inputs, encoded targets, and defaults used by the Prediction System. It is designed to support future improvements without changing the academic route structure.
