Dataset selection justification

Why these sources are appropriate for Machine Learning

The selected datasets represent the main dimensions required for recycling infrastructure analysis: population demand, social vulnerability, facility availability, recyclable potential, hazardous waste pressure, and demographic growth. This mix allows SmartRecycleAI to model the problem with numerical and categorical evidence rather than relying on a single administrative indicator.

Numerical Variables

Population, density, waste kilograms, collection coverage, distances, capacity, growth, and poverty indicators support statistical modeling.

Categorical Variables

Municipality, department, source labels, priority classes, and facility status support grouping, interpretation, and target construction.

Real-world Context

Every source contributes to a public planning question: where recycling infrastructure should be strengthened or created.

EDA interpretation

How to read the exploratory analysis

EDA element Purpose Technical interpretation
Missing value analysis Identify incomplete sources and decide cleaning strategy. Missing rates guide imputation, exclusion, and quality warnings before model training.
Distribution charts Understand skew, extreme values, and scale differences. Skewed waste and capacity variables justify scaling and robust interpretation.
Correlation analysis Detect relationships among demand, access, vulnerability, and infrastructure indicators. Correlations support feature review but do not prove causality.
Scatter analysis Connect poverty and access conditions visually. Relationships help explain why social indicators matter in recycling planning.

Data Quality Evaluation

Data quality is evaluated through row counts, column counts, variable types, missing values, duplicate records, source-specific quality notes, and relationship checks. The project avoids silent assumptions by showing these profiles directly in the Flask application.

This matters for Machine Learning because poor source quality can produce unstable features, unreliable model metrics, and misleading recommendations.

Data Limitations

Public datasets may use different reporting cycles, naming conventions, formats, and geographic granularity. Some records require normalization before joins, and not every variable represents a direct causal factor.

SmartRecycleAI treats these limitations as part of the academic analysis and connects them to the engineering phase where cleaning and transformations are documented.
