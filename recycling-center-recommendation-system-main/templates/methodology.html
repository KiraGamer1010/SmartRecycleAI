Process alignment

How CRISP-ML is implemented

Phase Implementation in SmartRecycleAI Academic evidence
Business Understanding Problem framing, expected value, feasibility, and risk analysis. Dedicated business page with objectives, questions, limitations, and impact.
Data Understanding Dataset catalog, EDA, missing values, distributions, correlations, and relationships. Generated charts, statistical tables, and documented data source contributions.
Data Engineering Reusable Python pipeline for cleaning, integration, transformations, encoding, and scaling. Processed CSV artifacts and script explanations tied to each preprocessing file.
Model Engineering Algorithm selection, dataset split, training strategy, hyperparameters, validation planning, supervised classifiers, and K-Means clustering. Dedicated model engineering page plus K-Means analysis with selected features, scaling, elbow method, clusters, and centroids.
Model Evaluation Accuracy, precision, recall, F1-score, ROC curves, confusion matrices, risk analysis, and expert review for recommendation quality. Dedicated evaluation page with classification metrics, comparative tables, comparative charts, and best-model selection.
Prediction System Selected supervised model is loaded through the same preprocessing pipeline used during training. Dedicated prediction page with form validation, class probabilities, confidence score, and result interpretation.
Model Deployment Production integration through Flask, Render, future APIs, documentation, and operational protocols. Dedicated deployment page with a production flow for institutional or municipal use.
Monitoring and Maintenance Continuous supervision, model updating, degradation detection, and risk review. Dedicated monitoring page explaining data drift, concept drift, and evolving environmental variables.

Detailed methodology explanation

Why CRISP-ML is useful for SmartRecycleAI

CRISP-ML extends the traditional CRISP-DM mindset by adding stronger attention to model quality, operationalization, monitoring, and maintainability. This matters because a recycling recommendation model must be technically correct, understandable for teachers, and responsible enough to support real municipal planning conversations.

Methodological Control

Each project decision is mapped to a phase, preventing the application from becoming a collection of disconnected pages.

Quality Assurance

The process requires reproducible data preparation, consistent preprocessing, test-set evaluation, and visible evidence.

Operational Thinking

Deployment, prediction forms, monitoring, and maintenance are treated as part of the ML lifecycle, not as optional extras.

Comparison with traditional development

Why a Machine Learning project needs more than normal software phases

Traditional software focus CRISP-ML focus SmartRecycleAI application
Build features from fixed requirements. Define a measurable learning problem and validate assumptions with data. The project connects recycling planning needs with public municipal datasets.
Test whether screens and functions work. Test whether the model generalizes and whether preprocessing is reproducible. Evaluation uses held-out test data, metrics, charts, and reusable artifacts.
Deploy once the application is complete. Deploy with monitoring, retraining criteria, and model risk controls. Render compatibility, prediction error handling, and monitoring guidance are documented.
