Feature Engineering

The pipeline creates urbanization ratio, rurality ratio, collection gap, estimated uncollected recyclable waste, center capacity pressure, access need score, recycling facility availability, hazardous waste exposure, and department population growth variables.

These transformations improve ML quality because they convert raw measurements into decision-oriented signals: demand, accessibility, vulnerability, operational capacity, and environmental pressure.

Transformation documentation

Machine Learning preparation decisions

Data engineering is the CRISP-ML phase where raw files become reliable model inputs. The project documents each transformation so a teacher can connect the original datasets with the supervised classifiers, K-Means clustering, generated charts, and prediction form.

Requirement Implementation ML reason
Data cleaning Source-specific parsing, normalized text keys, numeric conversion, and duplicate handling. Prevents inconsistent values from becoming false modeling patterns.
Null value handling Pipeline-level imputation and quality indicators are used before modeling. Allows models to train without hiding missingness problems from the academic report.
Categorical encoding Priority labels are encoded for classification, while descriptive labels remain visible for interpretation. Converts human-readable classes into model-compatible targets.
Feature engineering Creates access gap, pressure, capacity, growth, facility availability, and vulnerability indicators. Transforms raw measurements into decision-oriented predictors.
Normalization and scaling Model-ready features are scaled for algorithms that are sensitive to units and magnitude. Improves fair comparison among population, waste, distance, and capacity variables.

Feature Selection

Feature selection focuses on variables that directly describe recycling demand, social need, infrastructure access, facility capacity, environmental pressure, and demographic growth. These variables are understandable for academic review and relevant to the project objective.

Reproducibility

The transformation logic is centralized in the shared data pipeline, meaning charts, processed CSV files, Flask pages, K-Means artifacts, supervised model reports, and prediction metadata are derived from the same preparation process.
