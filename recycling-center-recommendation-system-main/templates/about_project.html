Architecture

The application uses a Flask application factory, a blueprint-based route module, reusable templates, static generated charts, and a centralized data pipeline. This keeps presentation, routing, and data engineering responsibilities separated.

Deployment Readiness

The project includes dependency management, a production WSGI entry point, Gunicorn startup support, a Procfile, and a Render service definition. Static charts are generated by the pipeline and served from the Flask static folder.

Final academic scope

What the repository is designed to demonstrate

The repository is the academic delivery package for SmartRecycleAI. It demonstrates a complete Machine Learning workflow inside a Flask application, including CRISP-ML explanation, dataset evidence, data engineering, model training, model evaluation, prediction, deployment readiness, and maintenance planning.

CRISP-ML Phases

Business Understanding, Data Understanding, Data Engineering, Model Engineering, Model Evaluation, and Prediction System are visible in the navbar.

System Capabilities

The application presents dashboards, charts, processed data, supervised models, K-Means analysis, and a functional prediction form.

Repository Purpose

The repository supports classroom review, local execution, GitHub delivery, Render deployment, screenshots, and final reporting.
